# Data Dictionary

## Project

AI vs Human Content Detection System

## Purpose

This document defines the dataset fields, engineered features, target variables, and deployment attributes used throughout the machine learning lifecycle.

---

# Source Dataset

## Dataset Name

ai_vs_human_text_2026.csv

## Total Records

2,000

## Classification Objective

Binary Classification

Target Classes:

| Value | Meaning       |
| ----- | ------------- |
| 0     | Human Written |
| 1     | AI Generated  |

---

# Original Dataset Fields

## text

### Description

Raw textual content used for classification.

### Data Type

String

### Example

```text id="uub1t6"
Artificial intelligence is transforming modern industries...
```

### Usage

Primary input feature.

---

## label

### Description

Ground truth classification label.

### Data Type

Integer

### Allowed Values

| Value | Meaning       |
| ----- | ------------- |
| 0     | Human Written |
| 1     | AI Generated  |

### Usage

Target variable for supervised learning.

---

## domain

### Description

Content category associated with the text sample.

### Data Type

Categorical

### Possible Values

* Academic
* News
* Social

### Usage

Exploratory analysis and validation.

Not used in final model training.

---

## source_model

### Description

Origin of generated content.

### Data Type

Categorical

### Examples

* Human
* GPT
* Gemini

### Usage

Validation analysis only.

Excluded from training to prevent data leakage.

# Engineered Features

The following features were created during Feature Engineering and Exploratory Analysis.

---

## word_count

### Description

Total number of words present in the text sample.

### Data Type

Integer

### Formula

Number of whitespace-separated tokens.

---

## char_count

### Description

Total number of characters present in the text sample.

### Data Type

Integer

---

## sentence_count

### Description

Number of detected sentences.

### Data Type

Integer

---

## avg_word_length

### Description

Average number of characters per word.

### Data Type

Float

### Business Interpretation

Measures vocabulary complexity.

---

## avg_sentence_length

### Description

Average words per sentence.

### Data Type

Float

### Business Interpretation

Measures writing structure complexity.

---

## unique_word_count

### Description

Number of unique words present.

### Data Type

Integer

---

## lexical_diversity

### Description

Ratio of unique words to total words.

### Formula

```text id="ekjlwm"
unique_word_count / word_count
```

### Data Type

Float

### Business Interpretation

Measures vocabulary richness.

# Final Production Features

The deployed model does not directly consume engineered numerical features.

Instead, it utilizes text vectorization methods.

---

## Word TF-IDF Features

### Method

TF-IDF Vectorizer

### Configuration

* N-Gram Range: (1,2)
* Maximum Features: 5,000
* Stop Words Removed
* Sublinear TF Enabled

### Purpose

Capture semantic word patterns.

---

## Character TF-IDF Features

### Method

Character TF-IDF Vectorizer

### Configuration

* Analyzer: char_wb
* N-Gram Range: (3,5)
* Maximum Features: 10,000
* Sublinear TF Enabled

### Purpose

Capture stylistic writing patterns.

# Decision Threshold

## Threshold Value

0.44

### Classification Logic

| Probability | Prediction    |
| ----------- | ------------- |
| < 0.44      | Human Written |
| ≥ 0.44      | AI Generated  |
